# 경기 상대 선택 범위 필터 설계

## 목표

일반 사용자가 경기 결과를 입력할 때 한 선수를 선택하면, 반대쪽 선수 목록에는 해당 선수 기준 위·아래 도전 범위 안에 있는 선수만 표시한다.

## 동작

- 아무 선수도 선택하지 않은 선택창은 전체 활동 선수를 순위순으로 표시한다.
- 한쪽 선수를 선택하면 반대쪽 선택창은 선택 선수 기준 활성 선수 순서로 위·아래 `challengeRange`명만 표시한다.
- 선택한 선수 본인은 상대 목록에서 제외한다.
- 선수 1과 선수 2 중 어느 쪽을 먼저 선택해도 같은 규칙을 적용한다.
- 기존 상대가 새 기준에서 범위를 벗어나면 해당 상대 선택을 초기화한다.
- 부상·비활동 선수는 기존 API와 동일하게 처음부터 선택 후보에 포함하지 않는다.

## 데이터 흐름

`GET /api/clubs/[club]/matches`가 활동 선수 목록과 해당 시즌의 `challengeRange`를 함께 반환한다. `MatchEntryDialog`는 API가 반환한 순위순 배열의 인덱스를 기준으로 상대 후보를 계산한다. 이 방식은 부상자를 제외한 활성 선수 순서로 범위를 계산하는 백엔드 `validateChallengeRange`와 일치한다.

## 안전성

프론트 필터는 잘못된 선택을 예방하는 UX다. 실제 경기 저장 시에는 기존 Supabase 트랜잭션의 도전 범위, 재대결 제한, 부상 상태 검증을 그대로 최종 권한으로 사용한다.

## 테스트

- GET 응답이 `challengeRange`를 포함하는지 검증한다.
- 선수 1 선택 후 선수 2 목록이 위·아래 범위로 줄어드는지 검증한다.
- 선수 2를 먼저 선택해도 선수 1 목록이 동일하게 줄어드는지 검증한다.
- API가 반환한 범위 값이 4가 아닌 경우에도 해당 값을 사용하는지 검증한다.
- 전체 테스트, 린트, 빌드와 모바일 브라우저 입력 흐름을 확인한다.
